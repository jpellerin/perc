Command: perc watch¶

Details¶

The watch command runs forever (in the foreground), waiting for changes to files in your application’s source and test directories. When it sees changes to application files, it runs perc test, perc build, and then perc check. When it sees changes to test files, it only runs perc test. If any command fails, further commands are not run until the next change is detected (and the failing command passes).

Usage¶

$ perc watch --help

  Usage: watch [options]

  Options:

    -h, --help      output usage information
    -T, --no-test   Do not run tests when application or test files change
    -B, --no-build  Do not build when application files change
    -C, --no-check  Do not run check after building

Options¶

-T, --no-test¶

Don’t run perc test when application or test files change.

-B, --no-build¶

Don’t run perc build when application files change.

-C, --no-check¶

Don’t run perc check after building.
